## âš¡ Getting Started

Follow these steps to get your development environment up and running.

### Prerequisites

Ensure you have the following installed on your machine:

- [PHP](https://www.php.net/) (v8.2 or higher)
- [Composer](https://getcomposer.org/)
- [Node.js](https://nodejs.org/) & NPM
- A database server (MySQL, PostgreSQL, or SQLite)

### ğŸ“¥ Installation

1.  **Clone the repository**

    ```bash
    git clone <repository-url>
    cd mini-crm
    ```

2.  **Install PHP dependencies**

    ```bash
    composer install
    ```

3.  **Install Node dependencies** (if valid for frontend assets)

    ```bash
    npm install
    ```

4.  **Environment Configuration**
    Copy the example environment file and configure your database settings.

    ```bash
    cp .env.example .env
    ```

    _Edit `.env` and set your database connection details (DB_HOST, DB_DATABASE, DB_USERNAME, DB_PASSWORD)._

5.  **Generate Application Key**

    ```bash
    php artisan key:generate
    ```

6.  **Run Migrations & Seed Database**
    This step sets up your database tables and populates them with initial test data (including the admin user).
    ```bash
    php artisan migrate --seed
    ```

## ï¿½ Running with Docker

Prefer using containers? We've got you covered. This is the easiest way to ensure your environment matches the production setup perfectly.

### Prerequisites

- [Docker](https://www.docker.com/) installed and running.

### ğŸš€ Docker Setup Steps

1.  **Clone and Configure**

    ```bash
    git clone <repository-url>
    cd mini-crm
    cp .env.example .env
    ```

    > **Important:** When running with Docker, update your `.env` file to match the container connection details:
    >
    > - `DB_CONNECTION=mysql`
    > - `DB_HOST=mysql`
    > - `DB_PORT=3306`
    > - `DB_DATABASE=crm_system`
    > - `DB_USERNAME=mini_crm`
    > - `DB_PASSWORD=mini_crm`

2.  **Start Containers**
    Build and start the services in the background.

    ```bash
    docker-compose up -d --build
    ```

3.  **Install Dependencies & Setup**
    Run the initial setup commands inside the application container.

    ```bash
    docker-compose exec app composer install
    docker-compose exec app php artisan key:generate
    docker-compose exec app php artisan migrate --seed
    ```

4.  **Access the Application**
    The API is now running at `http://localhost:8000/api`.
